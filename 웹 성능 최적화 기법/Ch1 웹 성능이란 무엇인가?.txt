1.1 웹

웹 : 인터넷이 제공하는 수많은 서비스 중 하나.
웹은 하이퍼텍스트를 바탕으로 관련 있는 문서끼리 연결한 문서의 집합체. 
하이퍼텍스트 : 텍스트를 뛰어넘는 다양한 컨텐츠를 아우르는 형식.

웹의 대표적인 요소
① URL : 클라이언트와 서버가 어떤 방식으로 데이터를 주고받을지 결정하는 프로토콜, 해당 서비스의 이름을 대표하는 도메인, 해당 자원이 위치하는 장소를 구조적으로 나눈 디렉터리, 실제 자원이 존재하는 파일 확장자를 포함하는 자원의 경로로 이루어짐.

② 네트워크 프로토콜 : 웹의 자원 위치에 접근하는 방식. 웹에서는 대부분 HTTP를 사용하는데 정보를 주고받는 헤더 부분과 실제 데이터를 주고받는 페이로드(바디) 부분으로 나뉘어 있음.

③ HTML : 다양한 웹상의 콘텐츠를 브라우저를 통해 나타낼 수 있는 태그. 마크업 언어. 


1.2 웹 성능이 중요한 이유

웹 성능이란 사용자가 원하는 서비스를 빠르게 전달받을 수 있도록 하는 시스템들의 성능을 의미. 즉, 웹 로딩 시간.

3초 법칙
① 웹 사이트의 로딩이 빨라야 한다.
② 웹 사이트의 머리말이 주목받을 수 있어야 한다.
③ 웹 페이지의 글이 눈에 띄어야 한다.
④ 웹 페이지 내 사용자 행동이 필요한 부분은 명확히 전달해야 한다.

페이지를 3초 안에 로딩되게 해야 한다.


1.3 웹 성능 측정 방법

웹 성능에 영향을 주는 요소
① 사용자 환경 - 각자의 네트워크 장비, 브라우저 환경이 다르다.
② 공급자 환경 - DNS 네임 서버 응답 속도, 웹 서버 응답 속도, 웹 사이트의 백엔드 처리 속도, 프론트엔드 최적화 여부
③ 전달 환경 - 웹 서버가 위치한 데이터 센터의 환경

웹 성능을 측정할 수 있는 서비스
① 크롬 개발자 도구
최적화를 진행할 경우 웹 로딩 시간이 빨라지는지 확인 가능, 폭포 차트(waterfall)는 어떤 콘텐츠 파일이 로딩 시간을 얼마나 소모했는지 시각적으로 나타냄.
② WebPageTest 서비스
웹 사이트 로딩 속도를 테스트할 수 있는 서비스.
③ 구글 PageSpeed
PSI라는 모듈을 이용해 웹 사이트의 성능 최적화 요소를 평가하는 서비스 제공.


1.4 웹 성능을 만드는 지표

* 사용자 환경 - 프론트엔드
- HTML로 웹 페이지 내용 구현, 자바스크립트가 웹 페이지 로직 담당, CSS로 UI 레이아웃 구성
- 빠르고 보기 쉽게 콘텐츠를 전달하는 것이 목적

* 공급자 환경 - 백엔드
- 사용자에게 보이는 프론트엔드 콘텐츠를 실제 생산하고 저장하여 네트워크를 통해 전달
- 네트워크 트래픽의 경로 설정, 서버 인스턴스 선택 등 다양한 네트워크 관련 결정 및 작업을 하는 라우터와 데이터베이스도 백엔드 요소
- 구글 애널리틱스로 측정해서 성능의 상태를 알 수 있다

* 전달 환경 - 네트워크
- 성능을 제약하는 두 가지 요소 : 대역폭, 지연 시간
- 대역폭 : 일정 시간에 처리할 수 있는 트래픽 양
- 지연 시간 : 기술적 이유로 사용자에게 콘텐츠를 전달하지 못하고 지연되는 시간
- 인터넷 서비스 사업자에 따라 대역폭과 지연 시간이 다르다.


1.5 웹 성능과 프론트엔드

대다수의 웹은 프론트엔드에서 가장 많은 시간을 소요함. 사용자 관점에서 웹 성능을 측정하기 때문에.


1.6 웹 성능 예산

성능 예산 : 웹 성능에 영향을 미치는 다양한 요소를 제어하는 한곗값. 웹 페이지의 파일 크기, 페이지 로딩 시간, 페이지에 포함된 이미지 파일 수 등등

성능 예산은 크게 세 분류로 나눌 수 있음.
① 정량 기반 지표
이미지, 스크립트, 폰트 등 웹 페이지 제작에 필요한 요소들에 대한 한곗값.
② 시간 기반 지표
정량 기반 지표의 단점을 보완하는 성능 예산.
브라우저에서 실제로 발생하는 다양한 웹 성능 이벤트 값을 측정하여 사용주가 느끼는 웹 성능에 대한 목표치를 설정하는 방식.
③ 규칙 기반 지표
웹 성능 측정 도구들이 웹 성능 지표를 측정하여 점수를 매기는데, 이 점수로 비교하는 것.
